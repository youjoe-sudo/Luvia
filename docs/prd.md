# منصة Luvia التعليمية - وثيقة متطلبات المنتج (PRD)

## 1. نظرة عامة على المنصة

### 1.1 اسم المنصة
Luvia

### 1.2 وصف المنصة
منصة تعليمية تربط صانعي المحتوى بالمتعلمين عبر نظام أكواد تفعيل فريد، مع التركيز على تجربة مستخدم مبسطة وآمنة، وتوفير محاضرات مباشرة تفاعلية.

### 1.3 الشعار
حرف L ثلاثي الأبعاد، يخرج منه مسارات طاقة تمثل انتقال المعرفة، بخلفية بيضاء سادة وألوان أزرق ملكي وبنفسجي كهربائي.

## 2. الوظائف الأساسية

### 2.1 نظام الأكواد (Voucher System)
- توليد أكواد فريدة لكل كورس
- كل كود صالح للاستخدام مرة واحدة فقط لكل مستخدم
- تتبع حالة الكود (مستخدم/غير مستخدم، من استخدمه، متى)
- إمكانية تحديد تاريخ انتهاء صلاحية للكود

### 2.2 نظام تسجيل الدخول والخروج
- تسجيل دخول بالبريد الإلكتروني وكلمة المرور
- إمكانية تسجيل الدخول عبر Google (اختياري، في مرحلة لاحقة)

### 2.3 عرض الكورسات
- صفحة رئيسية تعرض جميع الكورسات مع صور مصغرة وعناوين
- صفحة تفاصيل كل كورس (وصف، محتويات، سعر استرشادي، نبذة عن المحاضر)
- زرار تواصل عبر واتساب لشراء الكود تحت كل كورس (رابط مباشر للواتساب)

### 2.4 مشغل فيديو آمن (Secure Video Player)
- يتم تفعيل المشغل فقط إذا كان الكورس مملوكًا للمستخدم
- استخدام Google Drive لتشغيل الفيديوهات عبر Video ID فقط
- إضافة علامة مائية على الفيديو
- منع التنزيل منعاً باتاً
- منع لقطات الشاشة
- يمنع النقر بزر الفأرة الأيمن (Right-Click) على الفيديو
- يخفي الرابط الأصلي للفيديو لمنع التحميل المباشر
- يدعم تغيير جودة الفيديو وسرعة التشغيل
- حماية إضافية ضد الغش والسرقة:
  - كشف محاولات تسجيل الشاشة وإيقاف التشغيل تلقائياً
  - منع استخدام أدوات المطورين (Developer Tools) أثناء مشاهدة الفيديو
  - تشفير إضافي لبث الفيديو
  - مراقبة نشاط المستخدم أثناء المشاهدة
  - إيقاف التشغيل عند محاولة التبديل إلى نافذة أخرى أو تطبيق آخر
- زر إكمال الفيديو: يظهر زر إكمال الفيديو أثناء مشاهدة الدرس، ولا يمكن للطالب الوصول إلى الواجب الخاص بالدرس إلا بعد الضغط على زر إكمال الفيديو والانتهاء من مشاهدة الفيديو بالكامل

### 2.5 الدروس والمرفقات
- قائمة بالدروس داخل الكورس
- تحميل مرفقات الدرس (PDFs, Code Files) مباشرة من صفحة الدرس
- زر حل الواجب في نهاية صفحة الدرس
- شرط فتح الواجب: لا يتم فتح الواجب الخاص بالدرس إلا بعد إكمال مشاهدة الفيديو بالكامل والضغط على زر إكمال الفيديو
- عند الضغط على زر حل الواجب، تظهر واجهة حل الواجب للطالب تحتوي على:
  - عرض أسئلة الواجب بشكل واضح ومنظم
  - عرض خيارات الإجابة لكل سؤال (اختيار من متعدد - إجابة واحدة أو إجابات متعددة)
  - إمكانية تحديد الإجابات بسهولة
  - زر التسليم
  - عرض النتيجة مباشرة بعد التسليم مع تفاصيل الإجابات الصحيحة والخاطئة
- إظهار النتيجة مباشرة بعد التسليم

### 2.6 الامتحان الشامل
- الامتحان الشامل يظهر كدرس في قائمة الدروس
- يتم التعامل معه كدرس عادي في الترتيب والعرض
- عند الدخول إلى الامتحان الشامل، يتم عرض الأسئلة بدلاً من مشغل الفيديو
- يحتوي على أسئلة اختيار من متعدد (إجابة واحدة أو إجابات متعددة)
- يقوم الطالب بحل الامتحان والتسليم
- إظهار النتيجة مباشرة بعد التسليم

### 2.7 شهادات إتمام الكورس
- يقوم المدير بإنشاء الشهادات للطلاب المؤهلين
- توليد شهادة PDF: الشهادة تحتوي على:
  - اسم الطالب الثلاثي
  - وصف الشهادة
  - لوجو الكورس أو الفريق المقدم للكورس
  - توقيع المحاضر (يدوي، يكتب الاسم بخط صغير وجميل مناسب للتوقيعات)
  - QR Code للتحقق من صحة الشهادة
  - تاريخ الإصدار باللغة الإنجليزية
  - تصميم احترافي ومنسق
  - **تحديث موضع البيانات**: يتم رفع جميع البيانات الموجودة في أسفل الشهادة إلى الأعلى بشكل أكبر لتحسين التوزيع البصري والمظهر العام
- يمكن للمدير اختيار الطالب المستلم للشهادة عند إنشائها
- يمكن للطالب استلام الشهادة في نهاية الكورس من قسم الشهادات
- الشهادة بصيغة PDF قابلة للتحميل والطباعة

#### نظام التحقق من الشهادات
- **صفحة التحقق العامة**: صفحة متاحة للجميع (بدون تسجيل دخول) للتحقق من صحة الشهادات
- **آلية التحقق**:
  - يتم إدخال رقم ID الشهادة (Certificate ID) في حقل البحث
  - النظام يبحث في قاعدة البيانات عن الشهادة باستخدام الـ ID
  - إذا تم العثور على الشهادة، يتم عرض:
    - اسم الطالب الثلاثي
    - اسم الكورس
    - تاريخ الإصدار
    - وصف الشهادة
    - حالة الشهادة (صالحة / ملغاة)
    - رسالة تأكيد: هذه الشهادة صادرة من منصة Luvia وصالحة
  - إذا لم يتم العثور على الشهادة، يتم عرض رسالة: الشهادة غير موجودة أو غير صالحة
- **QR Code**: يحتوي الـ QR Code الموجود على الشهادة على رابط مباشر لصفحة التحقق مع الـ Certificate ID مدمج في الرابط
- **نسبة نجاح 100%**: النظام يعتمد على Certificate ID الفريد المخزن في قاعدة البيانات، مما يضمن دقة التحقق بنسبة 100%

### 2.8 المحاضرات المباشرة (Live Lectures)
#### للمعلم:
- إمكانية فتح محاضرة مباشرة مرتبطة بدرس معين
- نظام المحاضرات المباشرة يتضمن:
  - مشاركة الشاشة والصوت فقط
  - حظر المستخدمين من المحاضرة عند الحاجة
  - إنهاء المحاضرة وإغلاقها
  - حفظ المحاضرة بعد الانتهاء لرفعها على Google Drive لاحقاً
  - واجهة تحكم سهلة الاستخدام

#### للطلاب:
- الدخول إلى المحاضرة المباشرة من صفحة الدرس
- يشترط أن يكون الطالب مشترياً للكورس للدخول إلى المحاضرة
- فتح الميكروفون فقط (بدون كاميرا)
- إرسال رسائل في الدردشة الخاصة بالمحاضرة
- لا يمكن للطلاب الذين لم يشتروا الكورس الدخول إلى المحاضرة حتى لو كان لديهم حساب

### 2.9 قفل الجهاز (Device Lock)
#### عند أول تسجيل دخول:
- حفظ بصمة المتصفح (Browser Fingerprint) باستخدام تقنيات تحديد هوية الجهاز
- حفظ عنوان IP الخاص بالمستخدم

#### عند تغيير الجهاز أو المتصفح:
- عرض نافذة منبثقة للمستخدم تحتوي على رسالة: تم تسجيل دخول من جهاز جديد
- تسجيل محاولة الدخول من الجهاز الجديد

#### للأدمن:
- عرض جميع محاولات تسجيل الدخول من أجهزة جديدة
- إمكانية مراجعة تفاصيل كل محاولة (اسم المستخدم، الجهاز القديم، الجهاز الجديد، التاريخ والوقت)

## 3. واجهات المستخدم

### 3.1 واجهة الطالب (Learner Frontend)
- صفحة رئيسية: عرض الكورسات، شريط بحث، فلترة حسب الأقسام
- صفحة تفاصيل الكورس: وصف، قائمة الدروس (مقفلة)، نبذة عن المدرس، حقل إدخال كود التفعيل، زر WhatsApp
- صفحة كورساتي (My Courses): عرض الكورسات التي يمتلكها الطالب بشكل مختلف عن صفحة تفعيل الكورس
- صفحة الكورس المفعلّ: 
  - قائمة الدروس على اليمين (تتضمن الدروس العادية والامتحان الشامل كدرس)
  - الدرس المحدد يظهر على اليسار مع مشغل الفيديو
  - تحت الدرس يمكن رؤية الواجب (إن وجد)
- صفحة الدرس: مشغل الفيديو، زر إكمال الفيديو، المرفقات، زر حل الواجب في نهاية الصفحة (يتم تفعيله فقط بعد إكمال الفيديو)، زر الدخول إلى المحاضرة المباشرة (إن كانت متاحة)
- صفحة الامتحان الشامل: عرض أسئلة الامتحان، خيارات الإجابة، زر التسليم، عرض النتيجة بعد التسليم
- واجهة حل الواجب للطالب: 
  - عرض أسئلة الواجب بتصميم واضح ومنظم
  - عرض خيارات الإجابة لكل سؤال مع إمكانية التحديد
  - مؤشر تقدم يوضح عدد الأسئلة المجابة
  - زر التسليم
  - عرض النتيجة التفصيلية بعد التسليم (الدرجة، الإجابات الصحيحة والخاطئة)
- واجهة المحاضرة المباشرة للطالب: عرض شاشة المعلم وصوته، زر فتح/إغلاق الميكروفون، نافذة الدردشة لإرسال الرسائل، إشعار عند طرد الطالب من المحاضرة
- صفحة الشهادات: قسم جديد يعرض جميع الشهادات التي حصل عليها الطالب، مع إمكانية:
  - عرض قائمة بجميع الشهادات المتاحة للطالب
  - معاينة كل شهادة
  - تحميل الشهادة بصيغة PDF
  - عرض تفاصيل الشهادة (اسم الكورس، تاريخ الإصدار، وصف الشهادة)
- **صفحة التحقق من الشهادات**: صفحة عامة متاحة للجميع تحتوي على:
  - حقل إدخال لرقم ID الشهادة
  - زر التحقق
  - عرض نتيجة التحقق (تفاصيل الشهادة إذا كانت صالحة، أو رسالة خطأ إذا كانت غير موجودة)
  - تصميم بسيط وواضح يسهل الاستخدام
- صفحات أساسية: تسجيل الدخول، إنشاء حساب، إعادة تعيين كلمة المرور
- نافذة منبثقة لتسجيل الدخول من جهاز جديد: رسالة تنبيه تم تسجيل دخول من جهاز جديد

### 3.2 لوحة تحكم الإدارة (Admin Panel)

#### إدارة الكورسات:
- زر إضافة كورسات جديدة
- نموذج إضافة كورس يحتوي على:
  - حقل عنوان الكورس بالعربية
  - حقل عنوان الكورس بالإنجليزية
  - حقل وصف الكورس بالعربية
  - حقل وصف الكورس بالإنجليزية
  - حقل اسم المحاضر بالعربية
  - حقل اسم المحاضر بالإنجليزية
  - حقل السعر الاسترشادي
  - رفع صورة مصغرة للكورس
  - زر حفظ الكورس
- تعديل الكورسات الموجودة
- حذف الكورسات

#### إدارة الدروس داخل الكورسات:
- زر إضافة درس جديد
- نموذج إضافة درس يحتوي على:
  - قائمة منسدلة لاختيار الكورس
  - حقل عنوان الدرس بالعربية
  - حقل عنوان الدرس بالإنجليزية
  - حقل إدخال Google Drive Video ID لفيديو المحاضرة
  - زر رفع ملف PDF ملخص المحاضرة
  - زر حفظ الدرس
- تعديل الدروس الموجودة
- حذف الدروس
- إمكانية رفع ملخص PDF للمحاضرة أو الدرس

#### متابعة إكمال الفيديوهات:
- صفحة متابعة إكمال الفيديوهات
- جدول يعرض:
  - اسم الطالب
  - الكورس
  - الدرس
  - حالة إكمال الفيديو (مكتمل / غير مكتمل)
  - تاريخ ووقت الإكمال
- إمكانية فلترة حسب الكورس أو الطالب أو الدرس
- شريط بحث للبحث عن طالب أو درس معين

#### إدارة الواجبات:
- زر إضافة واجب جديد
- نموذج إضافة واجب يحتوي على:
  - قائمة منسدلة لاختيار الكورس
  - قائمة منسدلة لاختيار الدرس
  - حقل عنوان الواجب بالعربية
  - حقل عنوان الواجب بالإنجليزية
  - حقل وصف الواجب بالعربية
  - حقل وصف الواجب بالإنجليزية
  - حقل تاريخ التسليم
  - قسم إضافة الأسئلة:
    - إضافة أسئلة اختيار من متعدد يدوياً
    - تحديد نوع السؤال (اختيار من متعدد - إجابة واحدة / اختيارات متعددة - عدة إجابات صحيحة)
    - تحديد الإجابة الصحيحة أو الإجابات الصحيحة لكل سؤال
  - زر حفظ الواجب
- زر متابعة نتائج الطلاب في الواجبات
- صفحة متابعة النتائج تحتوي على:
  - جدول يعرض أسماء الطلاب الذين قاموا بالحل
  - الدرجات المحصلة لكل طالب
  - تاريخ التسليم
  - إمكانية عرض تفاصيل إجابات كل طالب
- إمكانية تعديل الواجبات الموجودة
- متابعة تقدم الطلاب في الواجبات

#### إدارة الامتحانات الشاملة:
- زر إضافة امتحان شامل للكورس
- نموذج الامتحان الشامل يحتوي على:
  - قائمة منسدلة لاختيار الكورس
  - حقل عنوان الامتحان بالعربية
  - حقل عنوان الامتحان بالإنجليزية
  - حقل وصف الامتحان بالعربية
  - حقل وصف الامتحان بالإنجليزية
  - حقل تاريخ التسليم
  - قسم إضافة الأسئلة:
    - إضافة أسئلة اختيار من متعدد يدوياً
    - تحديد نوع السؤال (اختيار من متعدد - إجابة واحدة / اختيارات متعددة - عدة إجابات صحيحة)
    - تحديد الإجابة الصحيحة أو الإجابات الصحيحة لكل سؤال
  - زر حفظ الامتحان
- ملاحظة: الامتحان الشامل يتم إضافته كدرس في قائمة الدروس، ويظهر للطالب كدرس عادي في الترتيب
- متابعة نتائج الطلاب في الامتحانات الشاملة
- إمكانية تعديل الامتحانات الموجودة
- متابعة تقدم الطلاب في الامتحانات

#### إدارة الشهادات:
- زر إضافة شهادة جديدة
- نموذج إضافة شهادة يحتوي على:
  - قائمة منسدلة لاختيار الكورس
  - قائمة منسدلة لاختيار الطالب المستلم للشهادة
  - حقل اسم الطالب الثلاثي
  - حقل وصف الشهادة
  - زر رفع لوجو الكورس أو الفريق المقدم للكورس
  - حقل توقيع المحاضر (نص يدوي بخط صغير وجميل مناسب للتوقيعات)
  - معاينة الشهادة قبل الحفظ
  - زر حفظ الشهادة
- سجل الشهادات الصادرة:
  - جدول يعرض جميع الشهادات الصادرة مع التفاصيل التالية:
    - رقم ID الشهادة
    - اسم الطالب
    - الكورس
    - تاريخ الإصدار
    - حالة الشهادة
  - أزرار إجراءات لكل شهادة:
    - زر عرض الشهادة: لمعاينة الشهادة بصيغة PDF
    - زر تحميل الشهادة: لتحميل الشهادة بصيغة PDF
    - زر تعديل الشهادة: لتعديل بيانات الشهادة (اسم الطالب، وصف الشهادة، اللوجو، التوقيع)
    - زر حذف الشهادة: لحذف الشهادة من النظام
  - شريط بحث للبحث عن شهادة معينة حسب اسم الطالب أو الكورس أو رقم ID الشهادة
  - إمكانية فلترة الشهادات حسب الكورس أو تاريخ الإصدار

#### إدارة الأكواد:
- زر توليد أكواد جديدة
- نموذج توليد كود يحتوي على:
  - قائمة منسدلة لاختيار الكورس
  - حقل تحديد تاريخ انتهاء الصلاحية (اختياري)
  - زر توليد الكود
- عرض الكود المولّد بعد التوليد مباشرة
- عرض الأكواد المتاحة والمستخدمة في جدول
- تتبع استخدام الكود (User ID, Timestamp)

#### إدارة المستخدمين:
- عرض قائمة الطلاب في جدول
- شريط بحث للبحث عن المستخدمين
- أزرار إجراءات لكل مستخدم:
  - زر حظر المستخدم
  - زر إيقاف الحساب
  - زر إرجاع الحساب
  - زر ترقية إلى مدير
- عند الضغط على أي زر، تظهر رسالة تأكيد قبل تنفيذ الإجراء
- إمكانية رؤية جميع المستخدمين وإدارتهم

#### مراقبة محاولات تسجيل الدخول من أجهزة جديدة:
- صفحة محاولات تسجيل الدخول
- جدول يعرض جميع محاولات تسجيل الدخول من أجهزة جديدة
- الأعمدة: اسم المستخدم، البريد الإلكتروني، بصمة المتصفح القديمة، بصمة المتصفح الجديدة، عنوان IP القديم، عنوان IP الجديد، تاريخ ووقت المحاولة
- شريط بحث للبحث عن محاولات معينة
- إمكانية فلترة المحاولات حسب التاريخ أو المستخدم

#### متابعة تقدم الطلاب:
- صفحة متابعة تقدم الطلاب في الدروس
- جدول يعرض:
  - اسم الطالب
  - الكورس المسجل فيه
  - عدد الدروس المكتملة
  - نسبة الإنجاز
  - آخر درس تم مشاهدته
- إمكانية فلترة حسب الكورس أو الطالب
- شريط بحث للبحث عن طالب معين

#### الإدارة العامة:
- إعدادات أرقام الواتساب:
  - حقل إدخال رقم واتساب افتراضي للمنصة
  - إمكانية تخصيص رقم واتساب مختلف لكل كورس
  - جدول يعرض الكورسات وأرقام الواتساب المخصصة لها
  - زر حفظ التغييرات
- زر إعادة ضبط المنصة:
  - عند الضغط على الزر، تظهر رسالة تحذيرية تطلب التأكيد
  - بعد التأكيد، يتم حذف جميع البيانات (الكورسات، الطلاب، الأكواد، الواجبات) مع الاحتفاظ بحساب المدير فقط

### 3.3 لوحة تحكم المدرس (Instructor Panel)

#### إدارة الكورسات:
- زر إضافة كورسات جديدة
- نموذج إضافة كورس يحتوي على:
  - حقل عنوان الكورس بالعربية
  - حقل عنوان الكورس بالإنجليزية
  - حقل وصف الكورس بالعربية
  - حقل وصف الكورس بالإنجليزية
  - حقل اسم المحاضر بالعربية
  - حقل اسم المحاضر بالإنجليزية
  - حقل السعر الاسترشادي
  - رفع صورة مصغرة للكورس
  - زر حفظ الكورس
- تعديل الكورسات الموجودة
- حذف الكورسات
- عرض قائمة بجميع الكورسات الخاصة بالمدرس

#### إدارة الدروس داخل الكورسات:
- زر إضافة درس جديد
- نموذج إضافة درس يحتوي على:
  - قائمة منسدلة لاختيار الكورس
  - حقل عنوان الدرس بالعربية
  - حقل عنوان الدرس بالإنجليزية
  - حقل إدخال Google Drive Video ID لفيديو المحاضرة
  - زر رفع ملف PDF ملخص المحاضرة
  - زر حفظ الدرس
- تعديل الدروس الموجودة
- حذف الدروس
- رفع فيديوهات المحاضرات عبر إدخال Google Drive Video ID
- رفع ملخصات PDF

#### إدارة الواجبات:
- زر إضافة واجب جديد
- نموذج إضافة واجب يحتوي على:
  - قائمة منسدلة لاختيار الكورس
  - قائمة منسدلة لاختيار الدرس
  - حقل عنوان الواجب بالعربية
  - حقل عنوان الواجب بالإنجليزية
  - حقل وصف الواجب بالعربية
  - حقل وصف الواجب بالإنجليزية
  - حقل تاريخ التسليم
  - قسم إضافة الأسئلة:
    - إضافة أسئلة اختيار من متعدد يدوياً
    - تحديد نوع السؤال (اختيار من متعدد - إجابة واحدة / اختيارات متعددة - عدة إجابات صحيحة)
    - تحديد الإجابة الصحيحة أو الإجابات الصحيحة لكل سؤال
  - زر حفظ الواجب
- زر متابعة نتائج الطلاب في الواجبات
- صفحة متابعة النتائج تحتوي على:
  - جدول يعرض أسماء الطلاب الذين قاموا بالحل
  - الدرجات المحصلة لكل طالب
  - تاريخ التسليم
  - إمكانية عرض تفاصيل إجابات كل طالب
- إمكانية تعديل الواجبات الموجودة
- حذف الواجبات

#### إدارة الامتحان الشامل:
- زر إضافة امتحان شامل
- نموذج الامتحان الشامل يحتوي على:
  - قائمة منسدلة لاختيار الكورس
  - حقل عنوان الامتحان بالعربية
  - حقل عنوان الامتحان بالإنجليزية
  - حقل وصف الامتحان بالعربية
  - حقل وصف الامتحان بالإنجليزية
  - حقل تاريخ التسليم
  - قسم إضافة الأسئلة
  - زر حفظ الامتحان
- ملاحظة: الامتحان الشامل يتم إضافته كدرس في قائمة الدروس
- متابعة نتائج الطلاب في الامتحانات الشاملة
- إمكانية تعديل الامتحانات الموجودة
- حذف الامتحانات

#### متابعة تقدم الطلاب:
- صفحة متابعة تقدم الطلاب في الدروس
- جدول يعرض:
  - اسم الطالب
  - البريد الإلكتروني
  - الكورس المسجل فيه
  - عدد الدروس المكتملة
  - نسبة الإنجاز
  - آخر درس تم مشاهدته
  - تاريخ آخر نشاط
- إمكانية فلترة حسب الكورس أو الطالب
- شريط بحث للبحث عن طالب معين
- عرض إحصائيات تفصيلية لأداء الطلبة

#### إدارة الطلاب:
- عرض قائمة الطلاب المسجلين في كورسات المدرس
- جدول يعرض:
  - اسم الطالب
  - البريد الإلكتروني
  - الكورسات المسجل فيها
  - تاريخ التسجيل
  - حالة الحساب (نشط / موقوف)
- شريط بحث للبحث عن الطلاب
- إمكانية فلترة الطلاب حسب الكورس
- عرض تفاصيل كل طالب عند النقر على اسمه

#### إدارة المحاضرات المباشرة:
- زر فتح محاضرة مباشرة
- نموذج فتح محاضرة مباشرة يحتوي على:
  - قائمة منسدلة لاختيار الكورس
  - قائمة منسدلة لاختيار الدرس المرتبط بالمحاضرة
  - زر بدء المحاضرة
- واجهة المحاضرة المباشرة للمعلم:
  - زر مشاركة الشاشة
  - زر فتح/إغلاق الميكروفون
  - قائمة بالطلاب المتصلين
  - زر حظر طالب من المحاضرة
  - نافذة الدردشة لعرض رسائل الطلاب
  - زر إنهاء المحاضرة
- بعد إنهاء المحاضرة:
  - خيار حفظ المحاضرة
  - إمكانية رفع المحاضرة المحفوظة على Google Drive لاحقاً

## 5. الميزات الإضافية

### 5.1 دعم اللغتين (Multilingual Support)
- المنصة بالكامل تدعم العربية والإنجليزية
- تبديل للغة في رأس الصفحة (Header)
- جميع النصوص الثابتة (Labels, Buttons) والمحتوى الديناميكي (عناوين ووصف الكورسات) متاحة باللغتين

### 5.2 الحماية من مشاركة الحساب
- تسجيل الـ User Agent أو IP Address عند أول تسجيل دخول للطالب
- عند اكتشاف محاولات تسجيل دخول متزامنة من أجهزة أو عناوين IP مختلفة بشكل متكرر، يتم تسجيل الخروج التلقائي أو إظهار تحذير
- الأدمن قادر على مراجعة هذه التنبيهات

### 5.3 قفل الجهاز (Device Lock)
- عند أول تسجيل دخول: حفظ بصمة المتصفح (Browser Fingerprint) الخاصة بالمستخدم، حفظ عنوان IP الخاص بالمستخدم
- عند تغيير الجهاز أو المتصفح: مقارنة بصمة المتصفح وعنوان IP الحاليين مع البيانات المحفوظة، إذا تم اكتشاف تغيير، يتم عرض نافذة منبثقة للمستخدم تحتوي على رسالة: تم تسجيل دخول من جهاز جديد، تسجيل محاولة الدخول من الجهاز الجديد في قاعدة البيانات
- للأدمن: صفحة خاصة لعرض جميع محاولات تسجيل الدخول من أجهزة جديدة، جدول يعرض: اسم المستخدم، البريد الإلكتروني، بصمة المتصفح القديمة، بصمة المتصفح الجديدة، عنوان IP القديم، عنوان IP الجديد، تاريخ ووقت المحاولة، إمكانية فلترة المحاولات حسب التاريخ أو المستخدم، شريط بحث للبحث عن محاولات معينة

### 5.4 التجاوبية (Responsiveness)
التصميم متجاوب تمامًا مع جميع أحجام الشاشات (كمبيوتر، تابلت، موبايل)

## 6. التقنيات المستخدمة
- الخادم (Backend): Node.js
- الواجهة (Frontend): React.js / Next.js
- قاعدة البيانات: تخزين الملفات في Database
- المحاضرات المباشرة: WebRTC أو خدمات مشابهة لتوفير بث مباشر آمن
